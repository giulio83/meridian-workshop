Request / Data Flow

1
User interaction / filter change
The global useFilters() composable holds shared reactive state (period, warehouse, category, status). Any view that calls getCurrentFilters() and watches the filter refs will re-fetch automatically.
2
API call via api.js
All HTTP calls go through client/src/api.js. Filter values are serialized to query params (e.g. ?warehouse=Tokyo&category=Sensors). Base URL is hardcoded to http://localhost:8001/api.
3
FastAPI endpoint filters in-memory data
On startup, server/mock_data.py loads all JSON files into module-level lists. Endpoints accept optional query params and filter the lists with Python list comprehensions. No ORM, no database queries.
4
Pydantic serialization → JSON response
Response models defined as Pydantic BaseModel subclasses. FastAPI serializes automatically. CORS is open (allow_origins=["*"]) for local development.
5
Vue reactive update
Views store raw data in ref() and derive display values in computed(). Currency formatting uses Intl.NumberFormat with the locale from useI18n().currentLocale.

AreaCurrent StateProduction Recommendation

AuthenticationNone — all endpoints public. User profile is mock data.Add OAuth2 / JWT via FastAPI security utilities
